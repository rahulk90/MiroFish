KEY INSIGHTS FOR TRANSLATORS
================================================================================

Common Chinese Patterns to Look For:
  1. Comments starting with "//", "/*", "<!-- "
  2. Error messages in addLog() calls
  3. Placeholder text in attributes
  4. UI labels in template span/div elements
  5. Status messages in conditionals
  6. Regex patterns with Chinese labels (CRITICAL)
  7. Object property keys with Chinese descriptions
  8. Default value strings

Translation Tips:
  - Keep technical terms consistent across all files
  - Platform names: "Info Plaza" (Twitter), "Topic Community" (Reddit)
  - Abbreviations: R=Round, T=Time, A=Actions, h=hours, m=minutes
  - Checkmark ✓ and X symbols for success/failure - keep as is
  - Tool names: Keep English prefix (InsightForge, PanoramaSearch, etc.)
  - Keep code variable names and function names unchanged
  - Keep API endpoint paths unchanged
  - Preserve all markdown formatting in rendered HTML
  - Maintain identical messaging between similar components

Content Sections to Pay Attention To:
  - User-facing messages (HIGH priority - affects UX)
  - Log messages (HIGH priority - affects debugging)
  - Error messages (HIGH priority - affects user support)
  - Comments (LOW priority - internal documentation)
  - Regex patterns (CRITICAL - affects functionality)
